# Tomcat Utils v1.2.0

> Powershell-скрипты для удобства развёртывания Java приложений разных версий Java, Maven и Tomcat.

# Первичная настройка
Директорию `tomcat_utils` можно разместить где-угодно.
> Я же избрал место по ближе к корню диска: `C:\workspace\tomcat_utils`.

Для автоматической настройки запустите скрипт [setup.bat](setup.bat) который вызовет [setup.ps1](bin/setup.ps1), выдав ему нужные права. 

Он внесёт все необходимые переменные окружения в PATH:
1. `%JAVA_HOME%\bin` - путь до jdk
2. `%CATALINA_HOME%\bin` - путь до tomcat
3. `%MAVEN_HOME%\bin` - путь до maven
4. `~\tomcat_utils\bin` - путь до `.ps1` скриптов

А так же он проводит первичную настройку всех tomcat экземпляров вызывая [setup_tomcat.ps1](bin/setup_tomcat.ps1)

> Локальные (пользовательские или User) `$JAVA_HOME` и `$CATALINA_HOME` Tomcat Manager создаёт и заполняет сам.

Если есть глобальные (системные или Machine) `$JAVA_HOME` и `$CATALINA_HOME` - удалите их.
В противном случае у вас будет путаница и вызов `catalina` будет давать разные результаты и часто завершаться ошибкой
```text
Error: Could not create the Java Virtual Machine.
Error: A fatal exception has occurred. Program will exit.
```
Однако даже так если вы будете переключать версии знайте - IDEA кеширует `PATH`
в момент запуска и изменения окружения не подтянутся корректно

> Поэтому если требуется вызвать `catalina` - лучше откройте `Powershell` отдельно, а не `Terminal` в интерфейсе IDEA

## Выдача разрешений
Если система блокирует исполнение `.ps1` скриптов выполните команду для предоставления разрешений на исполнение всем `.ps1`-скриптам:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## Конфигурирование Tomcat Utils
> Файл [tomcat_configs.json](tomcat_configs.json) содержит персональные и глобальные настройки для каждого экземпляра Tomcat

- TOMCAT_DIR - Указывает на корневую директорию всех экземпляров Tomcat. 
**Желательно чтобы все экземпляры Tomcat были расположены там.**

- JAVA_DIR - Указывает на корневую директорию всех экземпляров Java (JDK). 
**Желательно чтобы все JDK были расположены там.**
- MAVEN_DIR - Указывает на корневую директорию всех экземпляров Maven. 
**Желательно чтобы все JDK были расположены там.**

- MANAGER_USERS - список пользователей "по умолчанию", 
которые будут внесены в `%TOMCAT_HOME%\conf\tomcat-users.xml`. 
*Не затронет экземпляры Tomcat отсутствующие в файле [tomcat_configs.json](tomcat_configs.json)*

- JAVA_VERSIONS - Экземпляры Tomcat управляемые Tomcat Utils.
  - CONNECTORS - список **уникальных** портов используемые экземпляром Tomcat
  - SHUTDOWN_PORT - **уникальный** порт используемый для передачи команды `SHUTDOWN`
  - JAVA_HOME - полный путь до нужного экземпляра Java (JDK)
  - TOMCAT_HOME - полный путь до нужного экземпляра Tomcat
  - MAVEN_HOME - полный путь до нужного экземпляра Maven

> После каждого изменения этого файла рекомендуется вызывать скрипт [setup_tomcat.ps1](bin/setup_tomcat.ps1)


## Install Java Libs
> Скрипт выполняющий установку всех jar-либ из директории проекта `~/src\main\webapp\WEB-INF\lib`.
Возвращает в консоль XML-код - который надо вставить в `pom.xml` файл проекта.

> **ВАЖНО**: При каждом запуске устанавливает значения переменных окружения на те,
> что указаны в [tomcat_configs.json](tomcat_configs.json) для конкретной `JavaVersion`

- `JavaVersion` - обязательный параметр, определяющий какая конфигурация будет использоваться для установки либ.
- `GroupId` - текст в формате `you.group.id`, который будет использоваться в качестве `-DgroupId` аргумента при установке ВСЕХ либ из директории `~/src\main\webapp\WEB-INF\lib`.
- `Silent` - опциональный флаг, запуск с которым будет без вывода `[INFO]` сообщений, mvn логов и XML-вставки для `pom.xml`

```powershell
install_java_libs 1.8 my.group.id
```

## Tomcat Manager
> Основной скрипт для управления экземплярами Tomcat.

> **ВАЖНО**: Если экземпляр Tomcat был запущен без Tomcat Manager, 
> то он может занимать один или несколько требуемых поров - тем самым мешая запуску.
> *Tomcat Manager не может остановить работу такого экземпляра*

> **ВАЖНО**: При каждом запуске устанавливает значения переменных окружения на те,
> что указаны в [tomcat_configs.json](tomcat_configs.json) для конкретной `JavaVersion`

- `JavaVersion` - обязательный параметр, определяющий какая конфигурация будет запущена.
- `Mode` - обязательный параметр, определяющий какое действие будет выполнено с экземпляром Tomcat
  - deploy 
    - выполняет сборку проекта `mvn clean package -U`
    - копирование `.war` файла из локальной `~/target` директории в директорию `%TOMCAT_HOME%\webapps`
    - завершает работу активного экземпляра Tomcat (если он был запущен Tomcat Manager)
    - запускает экземпляр Tomcat
  - start
    - завершает работу активного экземпляра Tomcat (если он был запущен Tomcat Manager)
    - запускает экземпляр Tomcat
  - stop
    - завершает работу активного экземпляра Tomcat (если он был запущен Tomcat Manager)

- `Silent` - опциональный флаг, запуск с которым будет без вывода `[INFO]` сообщений
